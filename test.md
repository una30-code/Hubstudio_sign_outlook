# 测试说明（Test）

> 测试策略、用例、通过条件与产物位置。  
> **当前范围**：`requirements.md` **Phase 1**。

## 1. 测试类型

| 类型 | 工具/方式 | 说明 |
|------|-----------|------|
| 手工冒烟 | 按 §2 用例执行 | Hubstudio 已启动、`.env` 已配置时运行 `main`（实现后） |
| 自动化 | pytest + Playwright（推荐） | 与 `src/` 一致；需在可连 CDP 的环境跑 |

## 2. 用例表（映射需求）

| ID | 名称 | 映射需求 | 前置条件 | 步骤摘要 | 期望结果 |
|----|------|----------|----------|----------|----------|
| TC-001 | 连接 Hubstudio | F-001 | Hubstudio 运行，CDP 地址有效 | 执行连接逻辑 | 成功获得 browser / context / page |
| TC-002 | 打开注册页 | F-002 | TC-001 通过 | goto `OUTLOOK_REGISTER_URL`，带超时 | 页面加载完成，当前 URL 符合注册页 |
| TC-003 | 验证注册页 DOM | F-003 | TC-002 通过 | 断言注册相关标题或按钮可见 | 断言通过 |
| TC-004 | 端到端成功路径 | F-001～F-003 | 同上 | 串联 TC-001～003 | 满足 `requirements.md` §6 全部条件；`success=True`；有 success 级别日志 |
| TC-005 | 连接失败（负例） | §7 | 故意错误 CDP | 执行连接 | `success=False`，不崩溃，有错误日志；（若实现约定）有截图或明确 message |

负例覆盖（超时、无注册元素）可在自动化中分段模拟，或在 `debug_log.md` 记录手工验证结果。

## 3. 产物与报告

- **失败截图**：运行时由程序写入项目根目录 `screenshots/`（与 `requirements.md` §4 一致）。
- **测试报告 / 日志索引**：建议放在 `test-results/`（如 pytest `--junitxml`、自研清单），与运行时 `logs/` 区分：`logs/` 为被测程序运行日志，`test-results/` 为测试框架产出。

## 4. 通过 / 发布门槛（Phase 1）

- **必须**：TC-004 在当前约定环境下通过（或等价手工签字），且与 `requirements.md` §6 **成功定义** 一致。
- **建议**：TC-005 至少验证失败路径不崩溃且 `success=False`。
