# 需求说明（Requirements）

> 本文档为当前阶段唯一需求来源；与实现冲突时以本文为准。

---

## 0. 总目标

通过配置驱动方式，完成以下分阶段流程：

1. phase-0：创建 Hubstudio 指纹浏览器环境
2. phase-1：生成 Outlook 注册所需用户信息
3. phase-2：启动环境并执行 Outlook 注册页基础交互流程

---

## 1. 分阶段需求

### 1.1 phase-0 指纹浏览器环境准备

目标：通过配置文件创建可用的 Hubstudio 指纹浏览器环境。

#### 功能要求

- 必须从配置文件读取多条环境创建参数，不允许硬编码。
- 必填项至少包含：
  - 环境名称
  - IP 信息（代理或网络配置）
  - 指纹参数（浏览器基础指纹配置）
- 创建成功后，需返回可用于后续启动的环境标识（如 profile_id / 环境名称等）。

#### 验收标准

- 提供完整配置时，环境创建成功，返回 success = True。
- 缺少必填项时，创建失败并返回明确错误信息。
- 关键步骤有日志记录（开始创建、参数校验、创建结果）。

### 1.2 phase-1 用户信息生成

目标：自动生成符合约束的用户信息，用于 Outlook 注册流程。

#### 功能要求

- 姓名：美国风格姓名（名 + 姓）。
- 生日：年龄范围 18-55 岁（含边界）。
- 账号：基于美国姓名生成，并包含 5 位随机数字。
- 密码：长度 10 位，且必须同时包含数字、小写字母、大写字母。

#### 验收标准

- 生成结果字段完整（姓名、生日、账号、密码）。
- 字段格式满足上述规则。
- 输出结构包含 success / message / data（或等价结构）。

### 1.3 phase-2 Outlook 邮箱创建（基础流程）

目标：在已创建的指纹浏览器环境中，完成 Outlook 注册页自动化基础交互。

#### 功能要求

- 启动 phase-0 创建的 Hubstudio 指纹浏览器环境。
- 自动打开 Outlook 注册页面。
- 执行基础页面交互流程：
  - 页面加载等待与超时控制
  - 自动点击（按流程需要）
  - 关键元素存在性检查

#### 验收标准

- 可成功获取 browser / context / page（或等价对象）。
- 注册页 URL 正确且页面加载完成。
- 页面出现注册相关关键元素（标题、输入框、按钮等之一）。
- 输出 success = True，并记录关键日志。

---

## 2. 输入与输出约定

### 输入

- Hubstudio 环境配置（来源：配置文件 / `.env`）
- Outlook 注册 URL
- phase-1 生成的用户信息（用于后续扩展阶段）

### 输出

- success: bool
- message: str
- data: dict（可选，包含环境信息、用户信息、页面状态等）
- logs: 关键步骤日志
- screenshot_path: str（失败时必须输出或可定位）

---

## 3. 非功能需求

### 稳定性

- 页面加载与关键步骤必须设置超时。
- 任一阶段失败时，流程不崩溃，返回可读错误信息。

### 可观测性

- 必须记录关键步骤日志与错误日志。
- 失败时必须截图，保存到项目根目录 `screenshots/`。
- 对外返回统一结果结构（success / message / error）。

### 安全

- 不在代码中硬编码账号、密码、token、cookie 等敏感信息。
- 配置统一从 `.env` 或受控配置文件读取（`.env` 不入库）。

---

## 4. 范围外（当前版本不做）

- 不提交最终注册（若触发合规或风控风险）。
- 不做账号批量管理与生命周期管理。
- 不实现与需求无关的浏览器环境高级参数优化。

---

## 5. 成功定义

满足以下条件即视为成功：

1. phase-0：可通过配置成功创建 Hubstudio 环境。
2. phase-1：可稳定生成符合规则的用户信息。
3. phase-2：可启动环境并打开 Outlook 注册页，关键元素校验通过。
4. 流程返回 success = True，且关键步骤有日志可追溯。

---

## 6. 失败定义

以下任一情况视为失败：

- 配置缺失或格式错误导致环境创建失败。
- 浏览器环境无法启动或无法连接。
- 页面无法打开、超时或 URL 不符合预期。
- 页面关键元素缺失，无法进入下一步。
- 抛出未处理异常，或返回结构不完整。
