# 变更后文档同步

## 提示词正文（复制以下）

```
我完成了一批代码或配置变更，请在仓库内对齐文档，**不要**把尚未实现的能力写进文档。

请按需更新（由我勾选要你动的项）：
- [ ] `tasks.md`：任务状态、备注；是否新增后续任务（如 T-009）。
- [ ] `design.md`：若实现与设计不一致，以已提交代码为准修订对应小节（保持自包含叙述）。
- [ ] `requirements.md`：仅当行为变化影响验收口径时修改，并追加变更记录。
- [ ] `test.md`：新增或调整用例、执行说明。
- [ ] `debug_log.md`：若修复了已知问题，追加一条简短记录（脱敏）。

变更摘要（我填写）：
- 修改的文件：
- 行为变化：
- 仍存风险：

范围：默认只改勾选的文档，不动 `src/`，除非我明确说一起改。

（可选）是否需要同步 `prompts/` 下与本次变更相关的提示词：【是/否】
```
